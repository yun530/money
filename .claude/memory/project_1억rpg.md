게임형 저축 웹앱 "1억 RPG" 기획 및 구현 완료.

**Why:** 저축에 동기부여가 필요한 20~30대를 타깃으로, RPG 레벨업 메커닉으로 지루한 저축 여정을 게임처럼 만들기 위해 시작.

**How to apply:** 이 프로젝트 관련 작업 시 기획안(금융.md)과 구현(index.html) 모두 확인하고 진행.

## 파일 위치
- `/Users/yun/금융/금융.md` — 기획안 (검토 후 개선된 버전)
- `/Users/yun/금융/index.html` — 구현 완료된 단일 HTML 파일

## 기술 스택
- Vanilla JS + CSS3 + HTML5 단일 파일
- localStorage (회원별 분리)
- SubtleCrypto SHA-256 비밀번호 해싱
- 외부 의존성 없음 (Google Fonts CDN 제외)

## 구현된 핵심 기능
- 회원가입/로그인 (SHA-256 해싱, 세션 유지)
- 레벨 시스템 10단계 (누적 저금액 기준, 레벨별 UI 테마 변화)
- EXP 시스템 (저금액 / 1000)
- 저금하기 (빠른 금액 버튼, 메모, EXP 즉시 반영)
- 레벨업 연출 (화면 플래시 + 파티클 + 모달)
- 영구 미션 6종 + 월별 챌린지 4종 (매월 초기화)
- 삭제 규칙: EXP 유지, 미션 카운트 롤백, 레벨 하락 없음
- 월별 목표 설정 + 달성 시 뱃지 자동 획득
- 뱃지 컬렉션 (영구/월별 구분)

## 논의된 결정사항
- 은행 API 연동: 현실적으로 불가 (금융결제원 오픈뱅킹은 사업자 대상). 직접 입력 방식 유지.
- 허위 입력 허용: 게임 컨셉이므로 본인 책임. 의식(ritual)으로서의 입력 행위에 가치 있음.
- 비밀번호 평문 저장 → SHA-256 해싱으로 개선
