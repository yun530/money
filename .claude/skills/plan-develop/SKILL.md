다음 아이디어/초안을 실행 가능한 기획안으로 디벨롭해줘.

$ARGUMENTS

## 디벨롭 방식

### 1단계 — 핵심 정의
- **문제**: 이 기획이 해결하는 핵심 문제를 한 문장으로
- **목표**: 성공하면 어떤 상태가 되는지 (측정 가능한 형태로)
- **타겟**: 누구를 위한 것인지 (구체적인 사용자 페르소나)

### 2단계 — 구조 확장
초안에서 언급되지 않았지만 반드시 있어야 할 요소들을 채운다:
- 핵심 기능 / 서비스 구성 요소
- 사용자 시나리오 (메인 플로우 1~2개)
- 수익 모델 또는 가치 전달 방식
- 경쟁 구도 / 차별화 포인트

### 3단계 — 실행 로드맵
- **MVP 범위**: 가장 먼저 만들어야 할 최소 버전
- **단계별 일정**: Phase 1 / 2 / 3 형태로 큰 그림
- **필요 리소스**: 인력, 기술, 예산 관점에서 필요한 것

### 4단계 — 리스크와 가정
- 이 기획이 성립하기 위해 참이어야 하는 핵심 가정 3가지
- 각 가정이 틀렸을 때의 대응 방향

### 5단계 — 다음 액션
지금 당장 해야 할 것 3가지 (검증, 리서치, 제작 등 구체적으로)

## 원칙

- 없는 내용은 "가정: ~" 형태로 명시하고 채운다 — 불완전한 입력도 최대한 발전시킨다
- 과도하게 이상적이지 않게 — 실제로 실행 가능한 수준으로
- 한국어로 답변
- 결과물은 누군가에게 공유할 수 있는 기획서 수준으로 작성
